# 作业手册（基于 VSCode）

本手册面向 **计算思维课程大作业** 的同学，主要由`Claude Sonnet 4.6`生成。

> **关于作品公开时间**  
> 你的仓库是**私有的**，只有你和助教可以访问。  
> 截止日期后，助教会统一触发 GitHub Pages 部署，届时你的作品将出现在课程展示主页上。  
> **开发期间请使用本地预览**（详见第四节），无需等待在线部署。

## 一、环境准备（只做一次）

### 1. 安装 VSCode 与 Git

- VSCode：<https://code.visualstudio.com/>
- Git：<https://git-scm.com/downloads>（macOS 自带可直接用）

### 2. 注册 github.com 账号

- 访问 <https://github.com/signup>
- 使用你**提交给助教的邮箱**注册，否则收不到项目邀请

### 3. 配置 Git 身份

在终端执行：

```bash
git config --global user.name "你的姓名"
git config --global user.email "你注册 github.com 的邮箱"
```

### 4. 设置 github.com 凭据（推荐 HTTPS + PAT）

1. 登录 github.com → 右上角头像 → **Settings → Developer settings → Personal access tokens → Tokens (classic)**
2. 新建一个 token：name 随意，Scope 勾 `repo`，有效期选 6 个月
3. 复制 token（形如 `ghp_xxxxxxxxxxxxxxxxxxxx`）
4. 第一次 `git push` 时，Git 会弹出凭据框：
   - Username: 你的 github.com 用户名
   - Password: **粘贴刚才的 token**
5. macOS / Windows / Linux 的凭据管理器会把它记住，下次不用再输入

## 二、接受邀请并克隆项目

### 1. 接受邀请

助教会通过 GitHub 向你发送**仓库 Collaborator 邀请**（不是加入组织的邀请）。
在注册邮箱里找到标题含 `invited you to collaborate` 的邮件，
点击 **Accept invitation**。

或者登录 github.com 后，直接访问你的仓库地址：

```
https://github.com/THU-Computational-Thinking-2026-Spring/学号-拼音-项目slug
```

### 2. 在 VSCode 里打开

**最省心的方式**（推荐）：

1. VSCode 命令面板（`Cmd+Shift+P`）→ `Git: Clone`
2. 输入仓库 URL（在 GitHub 项目主页绿色 **Code** 按钮 → HTTPS 复制）
3. 选择本地目录保存
4. 打开后 VSCode 会弹窗 **"This folder has recommended extensions"** → 点 **Install All**
   会自动装好 Live Server、Prettier、GitLens、中文语言包等

## 四、写代码 → 本地预览

把你的作业文件放到 `public/`（入口必须是 `public/index.html`）。

VSCode 里有**两种本地预览方式**，任选其一：

### 方式 A：Live Server（推荐，自动刷新）

1. 打开 `public/index.html`
2. 右键 → **Open with Live Server**（底部状态栏也有 Go Live 按钮）
3. 浏览器会自动打开 `http://127.0.0.1:5500/`，保存文件即自动刷新

### 方式 B：一键任务

按 `Cmd+Shift+P` → `Tasks: Run Task` → **🌐 本地预览（Python http.server）**
然后浏览器打开 `http://localhost:8000/`。

### ⚠️ 资源路径注意

请全部使用**相对路径**：

- ✅ `<link rel="stylesheet" href="./style.css" />`
- ✅ `<img src="./assets/logo.png" />`
- ❌ `<link rel="stylesheet" href="/style.css" />`（部署后会 404）
- ❌ `<script src="http://localhost:8000/foo.js" />`

如果用 Vite/React/Vue 打包：
- **Vite**：`vite.config.ts` 里加 `base: './'`，然后把 `dist/` 内容复制到 `public/`
- **CRA**：`package.json` 加 `"homepage": "."`，把 `build/` 内容复制到 `public/`

## 五、提交前检查

VSCode 命令面板 → `Tasks: Run Task` → **✅ 提交前检查**

脚本会检查：

- `metadata.json` 字段是否齐全（不含 TODO）
- `public/index.html` 是否存在
- `public/thumbnail.*` 是否准备
- `docs/` 下是否有 PDF

如果全部 ✅，就可以 push 了。

## 六、提交到 GitHub

### 方式 A：VSCode 源代码管理（推荐）

1. 左侧活动栏点 **源代码管理（Source Control）**
2. 看到改动后，点 `+` 暂存所有改动（或针对单个文件点 `+`）
3. 在上方输入框写提交说明，如 `feat: 快排算法 v1`
4. 点 **提交** 按钮
5. 点 **同步更改（Sync Changes）** 或 **推送（Push）**

### 方式 B：命令行

```bash
git add -A
git commit -m "feat: 快排算法 v1"
git push origin main
```

> push 成功后可在仓库主页的 **Actions** 标签页看到状态（截止前无部署任务，显示为空是正常的）。

## 七、提交设计文档

1. 把 `学号_姓名_设计文档.pdf` 放到 `docs/` 目录下


PDF 会跟着仓库一起被归档，助教评分时能直接下载。

## 九、常见问题

### Q1: 收不到仓库邀请邮件

请确认你提交给助教的 GitHub 用户名和邮箱正确。联系助教手动添加。

### Q2: push 后 Actions 页面是空的

这是正常现象。开发期间 deploy.yml 只会在截止后被助教触发，无需担心。

### Q3: 打开部署后的 URL，样式丢失、JS 报错

99% 是**绝对路径问题**。把 `/style.css` 全部改成 `./style.css`。

### Q4: 本地 Live Server 打开不是 public 下的内容

在 VSCode 里 `.vscode/settings.json` 已设置 `liveServer.settings.root: "/public"`。
如果没生效，直接右键 `public/index.html` → Open with Live Server。

### Q5: 我不小心把 `node_modules/` commit 了

```bash
git rm -r --cached node_modules
echo "node_modules/" >> .gitignore
git add .gitignore
git commit -m "chore: ignore node_modules"
git push
```

### Q6: 截止日期后还能改吗？

不能。助教会在截止后冻结 main 分支。请务必提前提交，并留出时间本地验证。
